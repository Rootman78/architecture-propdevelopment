# Задание 3. Внешние интеграции

## Требования к безопасности

1. Шифрование данных при передаче персональных данных по сети. Уровень защиты TLS не ниже 1.2
2. Шифрование данных при хранении в базе данных. 
3. Двухфакторная аутентификация 
4. Доступ из внешних источников с использование определенных ключей шифрования и с определенных IP адресов
5. Передача персональных данных на внешние источники запрещена
6. Периодическое резервное копирование базы с персональными данными
7. Доступ к приложению умного дома имеют столько собственники с соответствующими ролями
8. Выполнять работу с персональными данными могут только сотрудники имеющие необходимый уровень доступа оформленный в виде роли Active Directory

## Требования к аутентификации и авторизации собственников

Для необходимого уровня безопасности персональных данных мы планируем использовать двухфакторную аутентификацию в auth-service-1:

#### Первый вход в мобильное приложение

1. При входе в систему пользователь вводит телефон, как логин и пароль для регистрации. 
2. Для подтверждения направляется проверочный код при помощи sms на указанный телефон
3. В случае корректного ввода кода предлагается ввести PIN код и добавить распознавание при помощи отпечатка пальцев

#### Последующие входы в мобильное приложение
Пользователь вводит PIN код или отпечаток пальцев

#### В случае изменения персональных данных профиля 

В этом случае необходимо дополнительно подтвердить изменения при помощи PIN и отпечатка

## Требования к аутентификации и авторизации внешних систем

Для необходимого уровня безопасности персональных данных мы планируем использовать аутентификацию с использованием JWT токенов:

1. Внешнему сервису выдается логин пароль. После корректного ввода логина и пароля, создается JWT токен, через который будет идти дальнейшее взаимодействие
2. Внешним системам даются такие права, которые позволяют получать только результат распознавания собственника, исключают получение и модификацию персональных данных

## Алгоритм работы системы умный дом при взаимодействии с внешними системами

### Регистрация

Если собственник, зарегистрированный в системе,  захочет пользоваться системой "Умный дом", ему нет необходимость регистрировать отдельный аккаунт и устанавливать специальное приложение. Для входа в систему мы воспользуемся существующим модулем auth-service-1 в котором (в случае отсутствия) установим двухфакторную аутентификацию с использованием номера телефона.

### Личный кабинет

В случае успешной аутентификации мы попадаем в личный кабинет приложения "Витрина для собственников", в который добавим функционал сбора биометрических данных для домофона и ввода информации но автомобилю для работы шлагбаума. Введенные данные через приложение tenant-smart-app попадают в базу данных tenant-smart-db в зашифрованном виде.

### Работа сервиса на примере системы "Интеллектуальный домофон" 

1. Собственник заранее зарегистрировался и через приложение передал свои персональные данные в систему
2. Собственник подходит к домофону, который обслуживает провайдер услуг "Умный дом".  Система провайдера, подключенная к камере домофона, по заданному алгоритму находит лицо пользователя.
3. На основе найденного лица, внешняя система создает вектор признаков, отправляет в нашу систему приложению tenant-smart-app в зашифрованном виде.
4. Приложение сверяет хранящиеся у нас вектор признаков персональных данные с переданными. В случае если сходство более 70%, отправляем внешней системе сообщение, о том, что человек найден в базе данных (либо его там нет). На основе этого ответа внешняя система понимает, пускать человека или нет.

### Работа сервиса на примере "Интеллектуальный шлагбаум" 

Работа аналогична "Интеллектуальный домофон", на исключением того, что внешняя система считывает данные автомобиля - внешний вид(больше шансов исключить подмену) и номер государственной регистрации
